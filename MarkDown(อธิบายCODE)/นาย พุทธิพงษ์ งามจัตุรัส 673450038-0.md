# อธิบาย Hook ในโปรเจกต์

ไฟล์นี้แบ่งคำอธิบายเกี่ยวกับ Hook ออกมาเป็นไฟล์แยกต่างหาก เพื่อให้เห็นว่าแต่ละ Hook ทำหน้าที่รับผิดชอบอะไรในโปรเจกต์นี้

## 1. useState
### หลักการทำงาน
useState ใช้เก็บค่าใน component และให้ component rerender เมื่อค่าเปลี่ยน

### ตัวอย่างโค้ด
```js
const [search, setSearch] = useState('')
const [category, setCategory] = useState('ทั้งหมด')
const [cartOpen, setCartOpen] = useState(false)
```

### เหตุผลที่เลือกใช้
- เหมาะสำหรับสถานะง่าย ๆ ที่ต้องอัปเดตใน component
- ใช้จัดการค่าที่แสดงผลใน UI เช่น คำค้นหาและสถานะตะกร้า

### ข้อดี
- ใช้ง่ายและเข้าใจง่าย
- เหมาะกับ state ที่ไม่ซับซ้อน

### ข้อจำกัด
- ขอบเขตอยู่ภายใน component เท่านั้น
- ถ้าต้องใช้ร่วมกันหลาย component ควรใช้ Context หรือ custom hook

---

## 2. useEffect
### หลักการทำงาน
useEffect ทำงานหลังจาก component render เสร็จ เพื่อจัดการ side effect เช่น localStorage หรือ DOM

### ตัวอย่างโค้ด
```js
useEffect(() => {
  document.documentElement.setAttribute('data-theme', theme === 'night' ? 'night' : 'day')
}, [theme])
```

### เหตุผลที่เลือกใช้
- ใช้สำหรับ sync ข้อมูลกับ browser หรือสถานการณ์ที่ต้องทำหลัง render
- เหมาะกับการอัปเดตค่าเมื่อ state เปลี่ยน

### ข้อดี
- แยก logic side effect ออกจาก UI ได้ชัดเจน
- ทำให้แอปตอบสนองต่อการเปลี่ยนแปลงสถานะได้ดี

### ข้อจำกัด
- ถ้าใช้มากเกินไปอาจทำให้โค้ดซับซ้อน
- ต้องระวังปัญหา rerun ที่ทำให้เกิด loop

---

## 3. useReducer
### หลักการทำงาน
useReducer ใช้สำหรับ state ที่มีหลาย action และ logic ซับซ้อน โดยแยกการเปลี่ยนแปลงออกเป็น action กับ reducer

### ตัวอย่างโค้ด
```js
const [state, dispatch] = useReducer(cartReducer, initialCartState)

dispatch({ type: 'ADD_ITEM', payload: product })
```

### เหตุผลที่เลือกใช้
- ตะกร้าสินค้าต้องมีงานหลายอย่าง เช่น เพิ่ม ลบ เพิ่มจำนวน ลดจำนวน ล้างตะกร้า
- ทำให้ logic อยู่ในจุดเดียวและง่ายต่อการ debug

### ข้อดี
- จัดการ state ที่ซับซ้อนได้เป็นระเบียบ
- โค้ดอ่านง่ายและบำรุงรักษาง่าย

### ข้อจำกัด
- สำหรับ state ที่ง่าย ๆ อาจดูซับซ้อนเกินจำเป็น
- ต้องมีการออกแบบ action และ reducer อย่างชัดเจน

---

## 4. useMemo
### หลักการทำงาน
useMemo จะจำค่าที่คำนวณไว้และคำนวณใหม่เฉพาะเมื่อ dependency เปลี่ยน

### ตัวอย่างโค้ด
```js
const total = useMemo(
  () => state.items.reduce((sum, item) => sum + item.price * item.qty, 0),
  [state.items]
)
```

### เหตุผลที่เลือกใช้
- ลดการคำนวณซ้ำในแต่ละ render
- เหมาะกับค่าที่ต้องคำนวณจากข้อมูลใหญ่หรือซับซ้อน

### ข้อดี
- เพิ่มประสิทธิภาพ
- ลดการคำนวณที่ไม่จำเป็น

### ข้อจำกัด
- ไม่ควรใช้กับทุกอย่างเพราะอาจเพิ่มความซับซ้อน
- ถ้าค่าคำนวณง่าย ๆ อาจไม่คุ้มค่า

---
