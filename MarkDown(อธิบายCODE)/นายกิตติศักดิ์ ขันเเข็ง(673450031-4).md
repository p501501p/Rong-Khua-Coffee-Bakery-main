
ไฟล์นี้แบ่งคำอธิบายเกี่ยวกับ Hook ออกมาเป็นไฟล์แยกต่างหาก เพื่อให้เห็นว่าแต่ละ Hook ทำหน้าที่รับผิดชอบอะไรในโปรเจกต์นี้

## 1.useCallback
### หลักการทำงาน
useCallback สร้างฟังก์ชันใหม่เฉพาะเมื่อ dependency เปลี่ยน เพื่อให้ reference ของฟังก์ชันคงที่

### ตัวอย่างโค้ด
```js
const addItem = useCallback((product) => {
  dispatch({ type: 'ADD_ITEM', payload: product })
}, [])
```

### เหตุผลที่เลือกใช้
- เหมาะกับฟังก์ชันที่ส่งต่อให้ child component
- ช่วยลดการ rerender ที่ไม่จำเป็น

### ข้อดี
- เพิ่มประสิทธิภาพในการ render
- ทำให้ฟังก์ชันมีความเสถียร

### ข้อจำกัด
- ถ้าใช้เกินไปอาจทำให้โค้ดซับซ้อนขึ้น
- เหมาะกับสถานการณ์ที่ฟังก์ชันถูกส่งต่อเป็น prop เท่านั้น

---

##  2. useContext
### หลักการทำงาน
useContext ใช้เพื่ออ่านข้อมูลจาก Context ที่อยู่เหนือ component เพื่อให้ component ใช้ข้อมูลร่วมกันได้

### ตัวอย่างโค้ด
```js
const { theme, toggleTheme } = useTheme()
```

### เหตุผลที่เลือกใช้
- เหมาะสำหรับข้อมูล global เช่น ธีมของแอป
- ลดปัญหา props drilling

### ข้อดี
- ทำให้โค้ดสะอาดขึ้น
- ลดความซ้ำซ้อนของ props

### ข้อจำกัด
- เหมาะกับข้อมูลที่ใช้ร่วมกันทั่วทั้งแอปเท่านั้น
- ถ้าใช้กับข้อมูลที่มีขอบเขตจำกัด อาจยากต่อการติดตาม
